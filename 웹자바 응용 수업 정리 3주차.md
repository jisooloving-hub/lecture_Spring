## 2. 프로젝트 실행 과정

브라우저에서 `http://localhost:8080` 호출 시 다음 순서로 처리된다.

```
① 웹 브라우저
      ↓
② Tomcat: web.xml
   - DispatcherServlet이 웹 요청 URL을 제어
      ↓
③ Spring: servlet-context.xml
   - 컨트롤러 클래스를 검색
      ↓
④ Spring: HomeController.java
   - 웹 요청 URL을 처리한 후 결과를 출력할 뷰 이름을 DispatcherServlet에 반환
      ↓
⑤ Spring: servlet-context.xml
   - 컨트롤러에서 반환한 뷰 이름으로 뷰를 검색
      ↓
⑥ Tomcat + Spring: home.jsp
   - 처리 결과가 포함된 뷰를 DispatcherServlet에 반환하고 최종 결과를 브라우저에 전달
```

## 3. 웹 프로젝트 환경 설정 파일 (web.xml)

- **역할**: 웹 프로젝트의 배치 설명자(deployment descriptor)
- 웹 프로젝트가 배포되는 데 이용되는 XML 형식의 자바 웹 애플리케이션 환경 설정 담당
- Servlet 3.0 이상에서는 Annotation으로 대체 가능

### web.xml 주요 구성 요소

```xml
<!-- 서블릿 컨텍스트가 시작될 때 전달할 파라메터를 설정 -->
<context-param> ... </context-param>

<!-- 서블릿 컨테이너의 각종 이벤트를 수신/처리할 이벤트 리스너를 등록 -->
<listener> ... </listener>

<!-- 사용자의 요청을 처리할 서블릿을 등록 -->
<servlet> ... </servlet>

<!-- 서블릿을 사용자 요청 URL과 매핑 -->
<servlet-mapping> ... </servlet-mapping>

<!-- 서블릿 호출 전처리 필터를 등록 -->
<filter> ... </filter>

<!-- 필터를 요청 URL과 매핑 -->
<filter-mapping> ... </filter-mapping>
```

### 루트 컨텍스트 설정

- 스프링 애플리케이션 **전체에서 공유**할 수 있는 루트 컨텍스트 설정 파일의 위치를 지정
- 공통 Bean (Service, Repository(DAO), DB, log 등) 설정
- 주로 **View 지원을 제외한** Bean을 설정
- `<context-param>` 요소를 이용하여 설정 파일 위치 지정

```xml
<context-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>/WEB-INF/spring/root-context.xml</param-value>
</context-param>
<listener>
    <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>
```

### 서블릿 컨텍스트 설정

- DispatcherServlet이 모든 요청을 받을 수 있도록 설정
- DispatcherServlet은 요청 URL을 처리할 컨트롤러에 처리를 위임
- `<init-param>` 요소로 설정 파일의 위치 지정
- `<load-on-startup>`: 서블릿을 로딩하는 순서 설정 (여러 서블릿이 있을 때 로딩 순서 제어)

```xml
<servlet>
    <servlet-name>appServlet</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    <init-param>
        <param-name>contextConfigLocation</param-name>
        <param-value>/WEB-INF/spring/servlet-context.xml</param-value>
    </init-param>
    <load-on-startup>1</load-on-startup>
</servlet>
<servlet-mapping>
    <servlet-name>appServlet</servlet-name>
    <url-pattern>/</url-pattern>
</servlet-mapping>
```

## 4. Spring MVC 환경 설정 파일

### root-context.xml vs servlet-context.xml

| 파일 | 역할 |
| --- | --- |
| `root-context.xml` | • 다른 웹 컴포넌트들과 공유하는 자원들을 설정
• 뷰와 관련되지 않은 Service, Repository(DAO), DB 등 객체를 정의
• 공통 Bean을 설정하며 주로 뷰 지원을 제외한 Bean 객체를 설정 |
| `servlet-context.xml` | • 웹 요청을 직접 처리할 컨트롤러의 매핑을 설정(HandlerMapping)하거나
뷰를 어떻게 처리할지 설정(ViewResolver) |

> 디스패처서블릿 아래에 “서블릿 웹 애플리케이션 컨텍스트(컨트롤러, 뷰리졸버, 웹 관련 빈객체)”가 있고,
그 하위에 “루트 웹 애플리케이션 컨텍스트(서비스 중간 계층, 데이터소스 등)”가 위치한다.
> 

### root-context.xml 예시

```xml
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans" ...>
    <!-- Root Context: defines shared resources visible to all other web components -->
</beans>
```

### servlet-context.xml 주요 설정

```xml
<!-- 컨트롤러 매핑 설정: 어노테이션 기반으로 관리하도록 설정 -->
<annotation-driven/>

<!-- Bean 객체 자동 등록: 기준 패키지 하위의 모든 클래스 파일 검색하여 자동 등록 -->
<context:component-scan base-package="com.springmvc.*" />

<!-- 정적 리소스 파일들을 찾을 기본 경로 설정 -->
<resources mapping="/resources/**" location="/resources/" />

<!-- 뷰 매핑 설정 -->
<beans:bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
    <beans:property name="prefix" value="/WEB-INF/views/" />
    <beans:property name="suffix" value=".jsp" />
</beans:bean>
```

## 5. 컨트롤러 매핑 설정

### `<annotation-driven/>` 역할

- `@Controller`, `@RequestMapping` 같은 어노테이션이 선언된 클래스를 찾아 **자동으로 등록**
- 핸들러 매핑(`RequestMappingHandlerMapping`)과 핸들러 어댑터(`RequestMappingHandlerAdapter`)도 자동 등록

| 클래스 | 역할 |
| --- | --- |
| `RequestMappingHandlerMapping` | 웹 요청 URL과 컨트롤러를 매핑, 어떤 컨트롤러가 처리할지 결정 |
| `RequestMappingHandlerAdapter` | 핸들러 매핑에 의해 결정된 컨트롤러를 호출, 응답 결과를 DispatcherServlet이 요구하는 형식으로 변환 |

---

## 6. 자바 클래스의 Bean 객체 설정

- XML에 등록하지 않고 설정된 패키지 하위의 모든 클래스를 검색하여 자동 등록

```xml
<context:component-scan base-package="com.springmvc.*" />  or
<beans:bean class="com.springmvc.controller.HomeController"/>
```

### 자동으로 감지하는 어노테이션

`@Component`, `@Repository`, `@Service`, `@Controller`, `@RestController`, `@ControllerAdvice`, `@Configuration`

### 구성요소 클래스에서 자동으로 활성화하는 어노테이션

`@Required`, `@Autowired`, `@PostConstruct`, `@PreDestroy`, `@Resource`, `@PersistenceContext`, `@PersistenceUnit`

## 7. 정적 리소스 설정

- **정적 리소스**: 브라우저의 요청 리소스가 이미 만들어져 있어 그대로 응답 가능한 자원 (이미지, js, css 파일 등)
- 캐시 헤더와 함께 효율적으로 제공하기 위한 핸들러 구성
- Spring의 리소스 처리를 통해 도달 가능한 모든 경로에서 리소스를 제공

```xml
<resources mapping="/resources/**" location="/resources/" />
```

### `<resources>` 요소 속성

| 속성 | 설명 |
| --- | --- |
| `mapping` | 웹 요청 URL 경로 패턴 설정 (컨텍스트 경로를 제외한 나머지 경로와 매핑) |
| `location` | 실제 자원의 위치 설정 (여러 곳이면 쉼표로 구분) |
| `cache-period` | 웹 브라우저에 캐시 시간 관련 응답 헤더를 전송 (초 단위, 0이면 캐시 안 함) |

## 8. 뷰 매핑 설정

- 브라우저에 요청 결과를 전달할 View 관련 설정

```xml
<beans:bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
    <beans:property name="prefix" value="/WEB-INF/views/" />
    <beans:property name="suffix" value=".jsp" />
</beans:bean>
```

### 뷰 이름 → 실제 파일 경로 변환

```
접두사(prefix) + View 이름 + 접미사(suffix) = 실제 View의 위치

예시: /WEB-INF/views/ + home + .jsp = /WEB-INF/views/home.jsp
```

컨트롤러에서 `return "home";` 반환 시, 위 규칙에 따라 `/WEB-INF/views/home.jsp`를 렌더링한다.

## 9. Maven 설정 (pom.xml)

### Maven 이란?

- Java 프로젝트들을 위한 **빌드 자동화 도구**
- 소프트웨어의 종속성과 빌드 방법을 관리

| 기능 | 설명 |
| --- | --- |
| 종속성 관리 | 사용할 라이브러리와 라이브러리가 참조하는 다른 라이브러리까지 자동 다운로드 |
| 빌드 관리 | 프로젝트를 배포 형식(jar, war)에 맞게 패키징, 종속성이 있는 라이브러리는 패키징 시 포함시킴. |
- `pom.xml` = POM(Project Object Model), XML 형식으로 Maven 프로젝트 설정

### pom.xml 주요 구성

**프로젝트 정보**

```xml
<groupId>webjava</groupId>
<artifactId>book-store</artifactId>
<version>0.0.1</version>
<packaging>war</packaging>
```

**속성 정보**

```xml
<properties>
    <java-version>25</java-version>
    <maven.compiler.target>25</maven.compiler.target>
    <maven.compiler.source>25</maven.compiler.source>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
</properties>
```

**의존성 라이브러리 정보**

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework</groupId>
        <artifactId>spring-webmvc</artifactId>
        <version>6.2.16</version>
        <scope>compile</scope>
    </dependency>
</dependencies>
```

| 태그 | 설명 |
| --- | --- |
| `groupId` | 라이브러리가 속한 그룹 |
| `artifactId` | 라이브러리의 ID |
| `version` | 라이브러리의 버전 |
| `scope` | 라이브러리가 사용되는 범위 |

> Maven은 설정된 종속성 라이브러리를 기본적으로 Maven Central 사이트를 통해 다운로드한다. (https://mvnrepository.com)
> 

### Maven Scope (종속성 범위)

종속성의 전이성을 제한하고, 종속성이 클래스 경로에 포함되는 시점을 결정한다.

| 범위 | 설명 |
| --- | --- |
| `compile` | 기본 범위. 프로젝트의 모든 클래스 경로에서 사용 가능, 종속 프로젝트로 전파됨 |
| `provided` | JDK 또는 컨테이너가 런타임에 종속성을 제공. 컴파일·테스트에만 추가, 런타임에는 미포함 |
| `runtime` | 컴파일에는 불필요하지만 실행에는 필요. 런타임 및 테스트 클래스 경로에만 포함 |
| `test` | 테스트 컴파일 및 테스트 실행 단계에서만 사용 |
| `system` | `provided`와 유사하나, 명시적으로 지정된 경로에서 JAR를 제공해야 함 |
| `import` | `<dependencyManagement>` 섹션의 pom 유형 종속성에서만 지원 |

## 10. 스프링 웹 MVC의 동작

### Controller 어노테이션

**`@Controller`**
- 이 클래스가 Spring MVC에서 Controller 역할을 담당한다는 것을 알려주는 어노테이션
- `<component-scan>`에 의해 Bean Container의 관리 대상이 됨

**`@RequestMapping`**
- 브라우저의 요청 URL과 메소드(클래스)를 매핑하는 어노테이션
- 브라우저 요청에 포함된 다양한 정보로 매핑 가능

| 속성 | 설명 |
| --- | --- |
| `value` (=`path`) | 매핑할 URL 경로 |
| `method` | HTTP 메소드 유형 (GET, POST 등) |
| `params` | 요청 파라미터 조건 |
| `headers` | 요청 헤더 조건 |
| `consumes` | 요청 Content-Type 조건 |
| `produces` | 응답 Content-Type 조건 |

> HTTP Method 유형에 따라 하위 어노테이션 사용 권장:
`@GetMapping`, `@PostMapping`, `@PutMapping`, `@DeleteMapping`
> 

### Controller에서 JSP로 파라미터 전달

`Model` 객체를 통해 데이터를 JSP로 전달한다.

```java
@RequestMapping(value = "/", method = RequestMethod.GET)
public String home(Locale locale, Model model) {
    Date date = new Date();
    DateFormat dateFormat = DateFormat.getDateTimeInstance(DateFormat.LONG, DateFormat.LONG, locale);
    String formattedDate = dateFormat.format(date);
    model.addAttribute("serverTime", formattedDate);  // JSP로 데이터 전달
    return "home";  // View 이름 반환
}
```

JSP에서는 EL(Expression Language)로 모델 데이터를 출력한다.

```
<P>The time on the server is ${serverTime}.</P>
```
