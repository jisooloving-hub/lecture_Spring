## 3. Java 소개

### Java의 역사

- 썬 마이크로시스템즈의 **제임스 고슬링** 팀이 개발한 객체지향 프로그래밍 언어
- 1991년 그린 프로젝트로 시작 → **1995년** 공식 발표
- 초기에는 가전제품용으로 개발, 현재는 웹 애플리케이션 서버 분야에서 주로 사용
- 2009~2010년 **Oracle**이 썬 마이크로시스템즈 인수 → 소유권 이전

### 주요 연혁

| 연도 | 내용 |
| --- | --- |
| 2012 | Android에서 Java API 사용 관련 Oracle vs Google 분쟁 |
| 2016 | Oracle, JDK9 기반에서 브라우저 플러그인(Applet) 중단 |
| 2017 | Oracle, Java EE를 Eclipse Foundation에 제출 |
| 이후 | javax/java 상표 합의 실패 → **Jakarta EE**로 명칭 변경 |

> • Java EE: 버전 8까지 지원
• Jakarta EE: 버전 8부터 지원
> 

### Java 특징

- **Write Once, Run Anywhere**
    - 컴파일러로 **바이트 코드**로 컴파일
    - 바이트 코드는 OS별 **가상머신(JVM)**에서 실행
- Java API는 **JCP(Java Community Process)**를 통해 표준화
    - **JSR(Java Specification Request)**: Java 플랫폼에 제안된 표준 제안문서

## 4. Spring Framework 소개

### Spring Framework란?

- 대규모 자바 애플리케이션 개발을 신속하고 편리하게 해주는 **오픈소스 프레임워크**
- ’Spring’은 Spring Framework 생태계 전체를 의미하기도 함
- 이름의 의미: EJB(겨울)의 복잡성을 극복한 **새 세대의 기술 = 봄(Spring)**

### 사용하는 이유

- 엔터프라이즈급(기업용) 개발이 가능한 **경량 오픈소스 프레임워크**
- 동적인 웹 사이트 개발을 지원하는 다양한 서비스 제공
- 생산성과 품질이 우수하며, 유지보수 해결책도 제시

### 역사

- **2004년 3월**: 1.0 버전 공식 배포 (EJB2.x의 가벼운 대안으로 출발)
- Spring Framework 6.x 지원 환경:
    - Java 17+
    - Jakarta EE 9
    - Tomcat 10.1, Jetty 11, Undertow 2.3
- 수업 사용 버전: **Spring Framework 6.2.x** (7.0.x는 아직 불안정)

## 5. Spring Framework 핵심 특징

### ① POJO(Plain Old Java Object) 지원

- 특화된 인터페이스 구현이나 의존성 높은 클래스 상속 없이도 개발 가능한 **일반 자바 클래스** 사용
- 장점:
    - 코드가 단순해짐
    - 특정 DB/서버 없이도 테스트 가능 → **개발 속도 향상**
    - 매우 자유로운 객체지향 설계 가능

### ② 의존성 주입(DI, Dependency Injection)

- 객체 간의 의존 관계를 개발자가 직접 생성/제어하지 않음
- **외부 Bean 설정 파일 또는 어노테이션**을 통해 스프링 컨테이너가 자동 연결

**DI 구현 방식 비교**
| 방식 | 설명 |
|——|——|
| 기존 방식 | `new` 키워드로 개발자가 직접 객체 생성 |
| Setter 주입 | setter() 메서드를 통해 외부에서 객체 주입 |
| 어노테이션(`@Autowired`) | 어노테이션 설정으로 자동 주입 |

- 장점:
    - 설정 파일로 스프링 컨테이너가 자동 연결
    - 코드가 간단해지고 테스트 용이
    - 변경 시 확장성이 매우 좋음

### ③ AOP(Aspect Oriented Programming) 지원

- 핵심 기능에서 **부가적인 공통 관심사(Cross-Cutting Concerns)**를 분리하여 공통 모듈로 관리
- 장점:
    - 공통 기능을 소스코드와 분리하여 재사용 가능
    - 개발자는 **비즈니스 로직에만 집중** 가능
    - 중복 없이 공통 모듈 작성 가능
- 대표 사용 사례: 공통 로깅, 인증 처리, 권한 제어, 트랜잭션 관리

### ④ 트랜잭션(Transaction) 지원

- **트랜잭션**: 쪼갤 수 없는 최소 작업 단위 (하나라도 실패 시 전체 실패 처리)
- DB 연동 기술과 트랜잭션 서비스 사이의 **종속성 제거**
- 트랜잭션 처리를 위한 **일관된 방법** 제공
- DB 연동 기술이나 환경이 바뀌어도 트랜잭션 코드는 그대로 유지 가능

## 6. Spring Modules (Spring Projects)

| 모듈 | 설명 |
| --- | --- |
| **Spring Framework (Core)** | 핵심 컨테이너 및 DI 프레임워크 (Beans, MVC, Web 등) |
| **Spring WebFlux** | 완전한 비차단(non-blocking) 지원 Reactive 웹 프레임워크 |
| **Spring Boot** | 독립실행형 앱, 내장 서블릿 컨테이너, 자동 구성, XML 불필요 |
| **Spring Data** | 다양한 DB(관계형/문서/그래프)와 연동, 명명 규칙 기반 데이터 처리 |
| **Spring Security** | 인증, 권한 부여, API 보안 처리 |
| **Spring Integration** | 메시징을 통한 내/외부 애플리케이션 간 통합 |
| **Spring Batch** | 대량 레코드 처리를 위한 배치 프레임워크 |
| **Spring Cloud** | 클라우드 네이티브 앱 개발 지원 (서비스 검색, 회로 차단기 등) |
| **Spring Native** | Spring Boot 프로젝트를 네이티브 실행 파일로 컴파일 |

> **비차단(non-blocking)**: I/O 처리 시 스레드가 대기하지 않고 다른 작업을 수행하는 비동기 처리 방식
