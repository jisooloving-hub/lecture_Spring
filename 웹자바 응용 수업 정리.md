# 웹자바 응용 수업 정리

### 자바

: 객체지향 프로그램이 언어.

: 웹 애플리케이션 서버 분야에서 많이 사용.

**특징**

: Write Once, Run Anywhere

- 컴파일러를 이용하여 바이트 코드로 컴파일
- 바이트 코드는 OS별로 공급되는 가상머신(VM)에서 실행됨

### Spring Framework(스프링/스프링 프레임워크)

: 대규모 자바 애플리케이션 개발을 신속하고 편리하게 해주는 오픈 소스 프레임워크

: 의미

- EJB(겨울)의 복잡성을 극복한 새로운 세대의 기술이라는 의미로 봄(spring)을 상징.
- 필요에 따라 쉽게 확장이 가능해서 (침대 스프링 느낌)탄력적이라는 의미를 담고 있음.

: 사용하는 이유

- 개발의 복잡함을 제거하면서 엔터프라이즈급(기업용, 상업용)을 개발할 수 있는 경량 오픈소스 프레임워크임.
- 스프링은 동적인 웹 사이트 개발을 지원하는 여러 서비스를 제공
    
    여기서 동적인 웹 사이트란 사용자의 요청이나 데이터에 따라 화면 내용이 계속 바뀌는 웹사이트를 말함.
    
- 웹 애플리케이션을 훨씬 효율적으로 구현할 수 있어서 생산성과 품질이 우수함.
- 개발 이후에 유지보수의 해결책도 제시해 줌.

: 역사

- EJB2.x 에 대한 가벼운 대안 제공
    - Enterprise JavaBeans(현재는 JAkarta Enterprise Beans)
    - 분산 컴포넌트 아키텍처(서로 다른 서버나 시스템에 있는 컴포넌트들이 네크워크를 통해 함께 동작하는 구조).
    - EJB2.x는 개발/배포/관리가 어려움.
    - EJB3.x에서는 기존 2.x의 문제점이 많이 개선됨.
- 지속성, 보안, 통합, 클라우드 컴퓨팅 등을 포함한 일반적인 개발 과제를 단순하게 해결하는 프레임워크를 지향함.

: 특징

- 일반적인 자바 객체를 위한 POJO지원(POJO : EJB같은 특정기술에 묶이지 않은 순수한 자바 객체 - 그냥 일반적인 자바 클래스 등.)
- 객체 간의 결합도를 줄이는 의존성 주입(DI) 지원.
    - 결합도 : 객체들이 서로 얼마나 강하게 연결되어 있는가를 의미.
- 의존성 주입 : 객체가 직접 다른 객체를 만들지 않고 외부에서 넣어주는 방식.
    - 객체 간의 관계를 관리할 때 사용하는 기법.
    - 객체 또는 구성 요소 사이의 의존 관계를 직접 생성하거나 제어하지 않음.
    - 외부의 빈(Bean)설정 파일 또는 어노테이션(Annotation)을 활용해 스프링 컨테이너가 자동으로 연결해주는 방식.
- 공통 모듈을 재사용하기 위한 AOP 지원. (Aspect-Oriented Programming : 프로그램의 공통 기능을 따로 분리해서 관리하는 프로그래밍 방식).
- 일관성 있는 모듈의 트랜잭션 지원.
- 전자정부 표준 프레임워크의 기반 기술
    - 대한민국 공공기관의 웹 애플리케이션 개발 시 사용이 권장되는, 국가가 관리하는 공식 프레임워크.
