# 웹자바 응용 수업 정리 2주차

# 스프링 웹 MVC 프로젝트의 구조

---

## 1. 스프링 웹 MVC란?

스프링이 제공하는 **웹 애플리케이션 개발 전용 프레임워크**

- **MVC 패턴** (Model - View - Controller) 을 사용
- 모델, 뷰, 컨트롤러 사이의 의존 관계 및 처리 흐름을 **스프링 컨테이너가 관리**
- 스프링이 제공하는 많은 기능을 확장하여 웹 애플리케이션 구축 가능

---

## 2. MVC 구성 요소

| 구성 요소 | 설명 |
| --- | --- |
| **프런트 컨트롤러** (Front Controller) | 애플리케이션의 흐름을 관리. Spring Framework의 `DispatcherServlet` |
| **모델** (Model) | 애플리케이션의 데이터가 들어 있는 객체 |
| **컨트롤러** (Controller) | 애플리케이션의 비즈니스 로직을 포함하는 자바 클래스. `@Controller` 사용 |
| **뷰** (View) | 모델의 정보(데이터)를 특정 형식에 맞게 표현. JSP, Thymeleaf, Freemarker 등 사용 |

---

## 3. 스프링 웹 MVC 기본 구성 요소

| 구성 요소 | 설명 |
| --- | --- |
| **DispatcherServlet** | 클라이언트의 요청을 전달받아 컨트롤러에 전달하고, 컨트롤러가 반환한 결과를 뷰에 전달하여 응답을 생성하는 등 **모든 처리 흐름을 담당** |
| **HandlerMapping** | 클라이언트의 요청 URL에 대해 **어떤 컨트롤러가 처리할지를 결정** |
| **HandlerAdapter** | 핸들러 매핑 클래스에 의해 결정된 **컨트롤러를 호출** |
| **Controller** | 클라이언트의 요청을 처리한 뒤 결과를 반환. 응답 결과에서 보여줄 데이터를 모델에 담아 뷰로 전달 |
| **ModelAndView** | 컨트롤러가 처리한 **결과 정보 및 뷰 선택에 필요한 정보를 담는 객체** |
| **ViewResolver** | 컨트롤러의 처리 결과를 사용자에게 보여줄 **뷰를 결정** |
| **View** | 컨트롤러의 처리 결과 화면을 생성. JSP를 비롯한 다양한 뷰 템플릿 엔진 사용. 클라이언트에 요청 처리 결과를 전송 |

---

## 4. 뷰 템플릿 엔진 (View Template Engine)

개발자가 뷰 템플릿을 정의할 수 있는 소프트웨어 구성요소

- 템플릿은 **동적 콘텐츠에 대한 위치 표시자가 있는 HTML 또는 XML**로 개발됨
- 형식과 구조가 정해진 **동적 웹 페이지를 생성**

**동작 방식**

```
View Model  ──┐
              ├──▶  View Template Engine  ──▶  HTML
View Template ──┘
```

- `View Model` : Controller가 생성
- `View Template` : Scope를 통해 View Template Engine에 전달
- `View Template Engine` : 모델과 템플릿을 결합하여 최종 HTML 생성

---

## 5. 스프링 웹 MVC 동작 흐름

| 순서 | 동작 | 관련 구성 요소 |
| --- | --- | --- |
| ① | 클라이언트가 처리 요청(URL) 전송 | 클라이언트 (WebBrowser) |
| ② | 요청 URL과 매핑되는 Controller 검색 요청 | DispatcherServlet → HandlerMapping |
| ③ | 요청 URL과 매핑되는 Controller 반환 | HandlerMapping → DispatcherServlet |
| ④ | 처리 요청 전달 | DispatcherServlet → HandlerAdapter |
| ⑤ | 처리 요청 변환 실행 | HandlerAdapter → Controller |
| ⑥ | 처리 결과 응답 반환 | Controller → HandlerAdapter |
| ⑦ | ModelAndView 반환 | HandlerAdapter → DispatcherServlet |
| ⑧ | 응답 생성 View 검색 | DispatcherServlet → ViewResolver |
| ⑨ | 응답 생성 View 반환 | ViewResolver → DispatcherServlet |
| ⑩ | Model 전달 및 응답 생성 요청 | DispatcherServlet → View |
| ⑪ | JSP 응답 생성 후 클라이언트에 전달 | View → 클라이언트 |
