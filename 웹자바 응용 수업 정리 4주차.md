## 1. 도서 쇼핑몰 시작 페이지 만들기

### WelcomeController.java

```java
package com.springmvc.controller;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;

@Controller
public class WelcomeController {
    private final Logger log = LoggerFactory.getLogger(WelcomeController.class);

    @RequestMapping(value="/home", method=RequestMethod.GET)
    public String welcome(Model model) {
        this.log.debug("welcome");
        model.addAttribute("greeting", "Welcome to BookMarket");
        model.addAttribute("strapline", "Welcome to Web Shopping Mall!");
        return "welcome";
    }
}
```

## 2. 부트스트랩 CSS 적용

### 부트스트랩(Bootstrap) 개요

- 웹사이트를 쉽게 만들 수 있게 도와주는 CSS, JS 프레임워크
- **반응형 웹 개발 지원**: 브라우저 크기에 따라 다른 페이지를 표현
- 하나의 코드로 다양한 기기(휴대폰, 태블릿, 데스크탑)에서 작동하는 웹 페이지 개발 가능
- 반응형 웹: **Responsive Web Design (RWD)**
- 공식 사이트: https://getbootstrap.com/

### welcome.jsp에 부트스트랩 적용


## 3. 웹 애플리케이션의 계층적 구조

### 계층(Layer)적 구조란?

- 전체 애플리케이션을 **관심사에 따라 계층별로 분리**한 구조
- 각 계층이 **느슨하게 결합**됨
- 계층 간에 유연하게 동작할 수 있도록 애플리케이션을 구조화함
- 웹 애플리케이션을 효율적으로 개발하고, 개발 후 **유지 보수를 쉽게** 하기 위해 시스템 구조를 계층화하여 개발

### 계층 구조를 사용하지 않을 경우 발생하는 문제

- 코드 복잡성 증가
- 유연성 부족
- 중복 코드의 증가
- 낮은 확장성
- 유지 보수의 어려움

> **Layer vs Tier**: Layer는 논리적 분리, Tier는 물리적 분리를 의미
> 

### 계층 구조 구성

| 계층 | 별칭 | 역할 |
| --- | --- | --- |
| 프레젠테이션 계층 (Presentation Layer) | 표현 계층 | 사용자와의 최종 접점. 데이터 입력/출력, 요청 전달 및 결과 표시 |
| 서비스 계층 (Service Layer) | 비즈니스 계층 | 포괄적인 서비스 처리. 프레젠테이션↔︎퍼시스턴스 계층 연결 |
| 퍼시스턴스 계층 (Persistence Layer) | 데이터 액세스 계층 | DB/파일에 접근하여 데이터 처리 |
| 도메인 객체 (Domain Object) | — | 정보를 저장하는 데이터 모델 |

```
웹 브라우저
   ↕ 요청/응답
프레젠테이션 계층 (View ↔ Controller ↔ Model)
   ↕
서비스 계층 (Service Layer)
   ↕
퍼시스턴스 계층 (Persistence Layer)
   ↕
데이터베이스
```

※ 도메인 객체(Domain Object)는 모든 계층에 걸쳐 데이터 교환에 사용됨

## 6. 도메인 객체 (Domain Object)

### 개념

- 애플리케이션에서 다른 계층과 **상호 데이터 교환**을 위한 데이터 객체
- 데이터 모델로 필요한 **속성(클래스의 필드)** 들을 정의하고 각 속성에 `setter()`, `getter()` 메서드를 생성

### Book.java

```java
package com.springmvc.domain;

public class Book {
    private String bookId;       // 도서ID
    private String name;         // 도서명
    private int unitPrice;       // 가격
    private String author;       // 저자
    private String description;  // 설명
    private String publisher;    // 출판사
    private String category;     // 분류
    private long unitsInStock;   // 재고수
    private String releaseDate;  // 출판일(월/년)
    private String condition;    // 신규 도서 or 중고 도서 or 전자책

    public Book() {
        super();
    }

    public Book(String bookId, String name, int unitPrice) {
        super();
        this.bookId = bookId;
        this.name = name;
        this.unitPrice = unitPrice;
    }

    // getters and setters (Eclipse 기능으로 자동 생성)
}
```

## 7. 퍼시스턴스 계층 (Persistence Layer)

### 개념

- **데이터 액세스(접근) 계층**
- 특정 클래스에 `@Repository`를 선언하면 해당 클래스는 **저장소 클래스**라는 의미를 가짐

### BookRepository.java (인터페이스)

```java
package com.springmvc.repository;

import java.util.List;
import com.springmvc.domain.Book;

public interface BookRepository {
    List<Book> getAllBookList();
}
```

## 8. 서비스 계층 (Service Layer)

### 개념

- 애플리케이션이 제공하는 **포괄적인 서비스들을 처리**하는 계층
- **프레젠테이션 계층과 퍼시스턴스 계층 사이를 연결**
- 특정 클래스에 `@Service`를 선언하면 해당 클래스는 **서비스 클래스**라는 의미를 가짐

### BookService.java (인터페이스)

```java
package com.springmvc.service;

import java.util.List;
import com.springmvc.domain.Book;

public interface BookService {
    List<Book> getAllBookList();
}
```

### BookServiceImpl.java (구현 클래스)

```java
package com.springmvc.service;

// ...import 생략

@Service
public class BookServiceImpl implements BookService {
    @Autowired
    private BookRepository bookRepository;

    @Override
    public List<Book> getAllBookList() {
        return this.bookRepository.getAllBookList();
    }
}
```

## 9. 프레젠테이션 계층 (Presentation Layer)

### 구성 요소

| 구성 요소 | 설명 |
| --- | --- |
| **컨트롤러 (Controller)** | 브라우저의 요청을 처리하는 자바 클래스. `@Controller` 선언 |
| **뷰 (View)** | 요청 처리 결과를 출력하는 JSP 웹 페이지. JSTL 적용 |
| **모델 (Model)** | JSP 웹 페이지에 출력할 데이터 형식 |

> **JSTL**: Java Server Pages Standard Tag Library
> 

### BookController.java

```java
package com.springmvc.controller;

import java.util.List;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;
import com.springmvc.domain.Book;
import com.springmvc.service.BookService;

@Controller
public class BookController {
    @Autowired
    private BookService bookService;

    @RequestMapping(value="/books", method=RequestMethod.GET)
    public String requestBookList(Model model) {
        List<Book> list = bookService.getAllBookList();
        model.addAttribute("bookList", list);
        return "books";
    }
}
```

### books.jsp

## 10. 계층별 어노테이션 정리

| 계층 | 어노테이션 | 의미 |
| --- | --- | --- |
| 퍼시스턴스 계층 | `@Repository` | 저장소 클래스 |
| 서비스 계층 | `@Service` | 서비스 클래스 |
| 프레젠테이션 계층 | `@Controller` | 컨트롤러 클래스 |
| 의존성 주입 | `@Autowired` | 스프링이 자동으로 빈(Bean) 주입 |
